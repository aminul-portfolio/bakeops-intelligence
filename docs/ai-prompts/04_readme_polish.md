# 04 — Enhanced README / Documentation Polish Prompt

## When to Use

Use this when polishing README sections, docs, GitHub descriptions, case studies, reviewer walkthroughs, or LinkedIn-ready project wording.

## Enhanced Prompt

```markdown
# README / Documentation Polish Prompt

## Role

Act as my senior DA/DE/AE portfolio documentation editor.

Polish the pasted README or documentation section for hiring-manager credibility, technical clarity, and evidence-based positioning.

## Project Context

Project: [PROJECT NAME]

This project is a Python/Django data-product or analytics platform.

Project purpose:
[BRIEFLY DESCRIBE THE PROJECT]

Target roles:

- Analytics Engineer
- Data Engineer
- Data Analyst
- BI / Reporting Analyst
- FinTech analytics
- Python/Django data-product roles

## Task

Please polish the pasted section for UK DA/DE/AE portfolio credibility.

## Rules

- Keep it evidence-based.
- Do not invent live users.
- Do not claim production deployment unless true.
- Do not claim billing or subscriptions unless implemented.
- Do not claim external POS, Shopify, Square, broker, exchange, or third-party integrations unless implemented.
- Do not exaggerate project maturity.
- Keep commercial-readiness wording realistic.
- Emphasise Python, Django, analytics, KPI dashboards, ETL thinking, BI exports, data quality, metric governance, and reporting reliability where relevant.
- Keep tone professional, recruiter-friendly, and technically credible.
- Mark unknowns clearly or avoid the claim.

## Section to Polish

```markdown
PASTE README OR DOCUMENTATION SECTION HERE
```

## Required Output

Please return:

1. Polished version
2. Concise version
3. Hiring-manager version
4. GitHub-friendly version
5. Any overclaiming risk
6. Claims that need evidence
7. Suggested screenshot/caption support if relevant
8. Final recommendation: keep / revise / remove

## Style Requirements

Use clean Markdown.

Avoid hype.

Prefer precise phrases such as:

- implemented
- planned
- prototype
- demo data
- seeded data
- reviewer-ready
- CI-verified
- test-covered
- BI-ready exports
- data quality visibility
- metric lineage
- operational reporting

Avoid unsupported phrases such as:

- production-ready
- enterprise-grade
- real customers
- live SaaS
- fully automated
- integrated with X
- proven commercial traction
```
