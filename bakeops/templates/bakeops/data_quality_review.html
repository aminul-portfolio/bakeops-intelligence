Reviewer Path

Trace issue → impact → action.

The trust surface is designed so a reviewer can inspect why an issue matters before accepting the dashboard output.

1. Identify severity Warning and info labels make the review priority visible.
2. Check issue type Issue type explains whether the problem is stock, expiry, production, or allocation-related.
3. Read trust impact Trust impact explains how the issue affects analytical confidence.
4. Review action Suggested action keeps the page operational, not just technical.
